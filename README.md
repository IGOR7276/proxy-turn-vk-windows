**WDTT-PC** — это **форк** [amurcanov/proxy-turn-vk-android](https://github.com/amurcanov/proxy-turn-vk-android), адаптированный для **Windows**. Нативный Go-клиент поднимает локальный WireGuard-интерфейс на Windows, получает конфигурацию от вашего VPS и передаёт транспорт через TURN-серверы VK, маскируя соединение под обычный зашифрованный медиатрафик звонка.

> Оригинальный Android-клиент: [amurcanov/proxy-turn-vk-android](https://github.com/amurcanov/proxy-turn-vk-android)

## Возможности

- **Нативный Windows-клиент:** один exe без зависимостей (кроме wintun.dll)
- **Веб-интерфейс:** встроенная веб-морда с формой, профилями, логами и статистикой
- **CLI-режим:** запуск из командной строки со всеми параметрами
- **Автоматический WireGuard-интерфейс:** флаг `-windows-wg` создаёт WG-туннель на Windows
- **TURN/DTLS-транспорт:** получает временные TURN-учётные данные VK-звонка и поднимает DTLS-соединения к relay-серверу
- **WRAP/RTP AEAD:** внешний слой шифрования поверх DTLS/WireGuard, выводится из пароля через HKDF
- **Капча VK Smart Captcha:** Go v2-решатель + Auto WebView по цепочке
- **VK Auth fallback:** переключение между client_id при ошибках
- **DNS fallback:** Yandex DNS → системный DNS
- **Многопоточность:** до 108 воркеров, каскадный запуск групп

## Установка и запуск

### Быстрый старт (веб-интерфейс)

Скачай `wdtt-pc.exe` из [релизов](../../releases), запусти:

```powershell
.\wdtt-pc.exe -ui
```

Открой в браузере: `http://localhost:XXXXX` (порт покажется в консоли). Заполни поля и нажми **Подключиться**.

### CLI-режим

```powershell
.\wdtt-pc.exe -peer <VPS_IP>:56000 -vk <VK_HASH> -password <ПАРОЛЬ> -n 36 -windows-wg
```

### Параметры

| Флаг | Описание |
|------|----------|
| `-peer` | Адрес и порт VPS-сервера (обязательно) |
| `-vk` | Хеш VK-звонка (обязательно) |
| `-password` | Пароль подключения (обязательно) |
| `-n` | Количество воркеров (кратно 9, по умолчанию 36) |
| `-windows-wg` | Создать WireGuard-интерфейс на Windows |
| `-wg-interface` | Имя WG-интерфейса (по умолчанию WDTT) |
| `-client-ids` | ID клиентов VK через запятую |
| `-fingerprint` | Браузерный фингерпринт (chrome/safari/ios/android/firefox) |
| `-captcha-mode` | Режим капчи (auto/wv/rjs) |
| `-device-id` | ID устройства |
| `-turn` | Принудительный IP TURN-сервера |
| `-port` | Принудительный порт TURN-сервера |
| `-listen` | Локальный адрес (по умолчанию 127.0.0.1:9000) |
| `-ui` | Запустить веб-интерфейс |
| `-ui-port` | Порт веб-интерфейса (по умолчанию случайный) |

### Веб-интерфейс

При запуске с флагом `-ui` клиент поднимает локальный HTTP-сервер и открывает веб-морду:

- **Форма ввода:** все параметры подключения в удобном виде
- **Профили:** сохранение и загрузка настроек
- **Подключение/отключение:** одной кнопкой
- **Логи:** живой просмотр событий в реальном времени
- **Статистика:** активные воркеры, трафик, время работы

```
http://localhost:8080  (или случайный порт, указанный в консоли)
```

### Сборка

```bash
cd go_client
go build -o wdtt-client.exe -tags windows .
```

Требуется Go 1.21+.

## Деплой сервера

Исходный код сервера (`server.go`) и скрипт деплоя находятся в репозитории [amurcanov/proxy-turn-vk-android](https://github.com/amurcanov/proxy-turn-vk-android). Для работы WDTT-PC требуется запущенный сервер на VPS.

## Как это работает

```text
wdtt-pc.exe → wintun (WG интерфейс) → UDP 127.0.0.1:9000
    → WRAP RTP AEAD → TURN/DTLS → wdtt-server на VPS → интернет
```

1. Приложение запускает нативный Go-клиент `libclient.so` и передаёт ему адрес VPS, VK-хеши звонка, пароль туннеля и количество потоков.
2. Go-клиент выводит WRAP-ключ из пароля подключения, получает TURN-учётные данные через VK-звонок, при необходимости решает VK Smart Captcha и устанавливает DTLS-соединения через TURN relay.
3. Внешний WRAP-слой упаковывает DTLS-пакеты в RTP AEAD/ChaCha20-Poly1305, чтобы ключ не был захардкожен в APK и мог отличаться у каждого пароля.
4. Первый рабочий канал запрашивает у VPS WireGuard-конфигурацию через `GETCONF`, передавая локальный порт, `device-id` и пароль подключения.
5. Сервер проверяет пароль: главный пароль работает как владелец, сгенерированные пароли имеют срок действия и привязываются к первому устройству.
6. Android-часть парсит полученный WireGuard-конфиг, поднимает системный VPN-туннель и применяет исключения приложений.
7. Watchdog следит за Go-процессом, активными воркерами и сетевыми изменениями, перезапуская транспорт при сбоях.

## Получение VK-хеша

```text
VK → группа → звонок → ссылка приглашения → код после /join/
```

1. Откройте VK и создайте пустую группу или используйте существующую.
2. Начните групповой звонок.
3. Скопируйте ссылку приглашения.
4. Вставьте в WDTT всю ссылку или только хеш после `/join/`.
5. Можно использовать до **4 хешей** одновременно для распределения нагрузки и увеличения доступного числа потоков.

> [!IMPORTANT]
> При выходе из звонка нажимайте **«Просто завершить»**, а не **«Завершить для всех»**. Если закрыть комнату для всех участников, хеш перестанет работать.

## Деплой VPS

Серверная часть ставится через SSH:

```text
SSH → /tmp/deploy.sh + /tmp/wdtt-server → /usr/local/bin/wdtt-server
   → systemd wdtt.service → wdtt0 + NAT + firewall
```

По умолчанию используются:

- `56000/udp` — DTLS-сервер WDTT.
- `56001/udp` — внутренний WireGuard-порт сервера.
- `9000/udp` — локальный порт клиента.
- `10.66.66.0/24` — подсеть WireGuard-устройств.

Деплой сервера выполняется из Android-приложения (оригинальный репозиторий) или вручную по SSH.

## Управление доступом

WDTT-сервер поддерживает две модели подключения:

- **Главный пароль:** задаётся при деплое и используется владельцем сервера.
- **Сгенерированные пароли:** создаются при деплое или через Telegram-бота, имеют срок действия, привязываются к первому устройству и обновляют WRAP-ключи на сервере без перезапуска.

Один сервер может держать до **10 активных паролей**. Новые пароли генерируются длиной **16 символов**.

Команда `/list` показывает активные пароли и устройства. Через inline-кнопки можно отвязать устройство или удалить пароль. При удалении или истечении пароля соответствующий WRAP-ключ удаляется из памяти сервера.

## Требования к пользователю

- **Windows 10 или 11** (x64)
- **wintun.dll** — копируется автоматически из WireGuard for Windows, или положить вручную рядом с exe
- **VPS-сервер** с развёрнутым wdtt-server (см. [оригинальный репозиторий](https://github.com/amurcanov/proxy-turn-vk-android))
- **VK-аккаунт** для получения TURN-учётных данных (хеш звонка)

> [!NOTE]
> Для работы клиента не требуется установленный WireGuard for Windows. wintun.dll копируется автоматически из известных путей системы.

## Логирование

Веб-интерфейс показывает логи в реальном времени: получение VK-credentials, решение капчи, DTLS-handshake, WRAP-статус, готовность WireGuard, статистику активных воркеров и ошибки.

> [!NOTE]
> Если видны DTLS handshake, но `Активных: 0`, чаще всего указан неправильный пароль подключения. Если часть потоков уже активна, одиночные DTLS timeout не останавливают туннель.

## Форк и адаптация

Этот репозиторий — **форк** [amurcanov/proxy-turn-vk-android](https://github.com/amurcanov/proxy-turn-vk-android), адаптированный для Windows. Изменения относительно оригинала:

- Полностью переписан клиент под Windows (WireGuard-интерфейс через wintun)
- Добавлен веб-интерфейс с профилями, логами и статистикой
- Route exclusion для TURN-серверов (чтобы WG не ломал TURN-соединения)
- Keepalive sessCancel и sessionReadTimeout
- Исправлены ошибки подключения по новому API pion/turn v5
